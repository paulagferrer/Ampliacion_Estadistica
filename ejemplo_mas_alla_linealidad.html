Ejemplo para el tema: Más allá de la linealidad

En este documento vamos a simular datos sobre la relación entre la temperatura máxima diaria y la demanda de bicicletas compartidas en una ciudad.

La idea es construir un ejemplo donde la relación entre ambas variables sea claramente no lineal:

  • en días fríos la demanda es baja;
  • en días templados la demanda aumenta;
  • en días muy calurosos vuelve a disminuir.

Con estos datos compararemos los siguientes métodos:

  1. Regresión polinómica.
  2. Funciones escalonadas.
  3. Splines de regresión.
  4. Splines suavizados.
  5. Regresión local (LOESS).
  6. Modelos aditivos generalizados (GAM).

Usaremos un conjunto de entrenamiento y otro de prueba para comparar el rendimiento predictivo mediante el RMSE.

2 Generación de datos simulados

Supongamos que una ciudad dispone de un sistema de bicicleta compartida. Queremos estudiar cómo cambia el número de alquileres diarios en función de la temperatura máxima del día.

Para que el ejemplo sea realista, simulamos primero una secuencia anual de temperaturas y luego definimos una relación no lineal entre temperatura y demanda.

2.1 Simulación

set.seed(1234)

n <- 365
dia <- 1:n

# Temperatura máxima diaria simulada con patrón estacional
temp <- 18 + 10 * sin(2 * pi * (dia - 80) / 365) + rnorm(n, sd = 4)
temp <- round(pmin(pmax(temp, 0), 40),1)

# Función verdadera (desconocida en un problema real):
# la demanda sube con el buen tiempo y cae cuando hace demasiado calor.
mu <- 250 + 
  2400 * plogis((temp - 11) / 2.8) - 
  1700 * plogis((temp - 31) / 2.2)

# Añadimos ruido aleatorio
alquileres <- round(mu + rnorm(n, sd = 170))
alquileres <- pmax(alquileres, 50)

datos <- tibble(
  dia = dia,
  temp = temp,
  alquileres = alquileres,
  media_real = mu
)

5 Funciones escalonadas

Las funciones escalonadas dividen la variable explicativa en intervalos y ajustan un nivel constante en cada tramo.

crear_cortes <- function(x, k) {
  br <- quantile(
    x,
    probs = seq(0, 1, length.out = k + 1),
    na.rm = TRUE
  )
  
  br <- unique(br)
  
  # Asegura que ningún valor de test quede fuera
  br[1] <- -Inf
  br[length(br)] <- Inf
  
  br
}

6 Splines de regresión

Los splines de regresión permiten modelar relaciones no lineales de forma flexible sin recurrir a un único polinomio global para todo el rango de la variable explicativa. La idea consiste en dividir el rango de la variable en varios tramos y ajustar en cada uno un polinomio de bajo grado. Estos tramos se unen en puntos llamados nodos, pero no lo hacen de forma brusca: se impone continuidad y suavidad en esos puntos, de modo que la curva final sea continua y visualmente natural.

En esta sección usaremos una base B-spline. En lugar de construir directamente la curva tramo a tramo, una B-spline representa la función ajustada como combinación lineal de varias funciones base suaves y locales. Esto hace que el ajuste sea más estable numéricamente y que pequeñas variaciones en una zona del eje \(x\) no afecten innecesariamente a toda la curva.

El parámetro que vamos a comparar aquí es el número de grados de libertad (df), que controla la flexibilidad del spline: - con pocos grados de libertad, la curva es más rígida; - con más grados de libertad, la curva puede adaptarse mejor a curvaturas complejas; - si usamos demasiados, existe riesgo de sobreajuste.

7 4. Splines suavizados

Los splines suavizados también permiten modelar relaciones no lineales, pero lo hacen de una forma distinta a los splines de regresión. En lugar de fijar explícitamente un número de nodos o unos grados de libertad concretos y ajustar después la curva, los splines suavizados buscan directamente una curva que logre un buen compromiso entre dos objetivos:

  • ajustarse bien a los datos observados;
  • mantener una forma suave, evitando oscilaciones innecesarias.

Ese equilibrio se controla mediante un parámetro de penalización. Si la penalización es pequeña, la curva sigue más de cerca a los datos y resulta más flexible. Si la penalización es grande, se penalizan las curvaturas excesivas y la curva se vuelve más lisa. En smooth.spline(), este equilibrio puede elegirse automáticamente mediante validación cruzada, de modo que el propio procedimiento selecciona el nivel de suavidad que mejor capacidad predictiva parece tener.

# Ajuste del spline suavizado usando validación cruzada

8 Regresión local (LOESS)

La regresión local, también conocida como LOESS (Locally Estimated Scatterplot Smoothing), es un método flexible para modelar relaciones no lineales sin imponer una forma funcional global concreta. En lugar de ajustar una única curva para todo el rango de la variable explicativa, LOESS ajusta pequeños modelos locales alrededor de cada punto del eje \(x\), utilizando únicamente las observaciones cercanas.

La idea es estimar la respuesta en un valor concreto de temperatura, el método selecciona una vecindad de observaciones próximas, asigna mayor peso a las más cercanas y ajusta con ellas un polinomio de bajo grado. Repitiendo este proceso a lo largo de todo el rango de la variable se obtiene una curva suave que puede adaptarse bien a patrones complejos.

El parámetro más importante en LOESS es span, que controla el tamaño de la vecindad usada en cada ajuste local:

  • un span pequeño utiliza pocos datos cercanos, por lo que la curva es más flexible;
  • un span grande utiliza una vecindad más amplia, por lo que la curva es más suave;
  • si span es demasiado pequeño, el modelo puede sobreajustarse;
  • si es demasiado grande, puede perder detalles importantes de la relación.

En este ejemplo compararemos varios valores de span para estudiar cómo cambia el ajuste y cuál ofrece mejor capacidad predictiva en el conjunto de prueba.
